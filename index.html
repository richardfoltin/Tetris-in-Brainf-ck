// Mi ez egyáltalán

A Claude Code egy ágens, nem egy chatablak

A legtöbb AI eszköz javasol — te másolod be a kódot. A Claude Code cselekszik: a terminálodban fut, hozzáfér a projekted fájljaihoz, és önállóan végigvisz egész feladatokat. Megérti a kérésed, tervet készít, megírja a kódot, lefuttatja, és ha hibázik, kijavítja magát.

A terminálban él

Ott dolgozik, ahol te is: git, npm, tesztek, build. Nincs külön felület, amibe ki-be kell másolgatni.

Érti az egész projektet

Magától megkeresi a releváns fájlokat, követi a kódbázis konvencióit, és összefüggéseiben gondolkodik — nem csak egy kódrészletet lát.

Végtelenül bővíthető

Hookok, skillek, MCP-szerverek és subagentek: a saját igényeidre szabod. Erről szól ez az oldal lentebb. ↓

// A lényeg

Miért más, mint amit eddig láttál

Négy dolog, ami miatt a Claude Code nem csak „egy újabb AI", hanem egy eszköz, amivel tényleg gyorsabban szállítasz.

Cselekszik, nem csak tanácsol

Megnyitja, szerkeszti, létrehozza a fájlokat, lefuttatja a teszteket, és git diffet csinál. Te a végeredményt nézed át, nem te gépeled be.

Sima magyar (vagy angol) szöveg

Nincs új szintaxis, amit meg kell tanulni. Elmondod, mit akarsz, ahogy egy csapattársnak mondanád. Ő kérdez vissza, ha valami nem világos.

Te tartod a kontrollt

Veszélyes művelet (fájl törlése, parancs futtatása) előtt engedélyt kér. Beállíthatod, mit tehet automatikusan és mit nem.

A te workflow-odra szabod

Beleépíted a saját szabályaidat (hookok), a csapatod tudását (skillek), bekötöd a többi eszközt (MCP), és párhuzamosítasz (subagentek).

Indulj el ennyiből

Kell hozzá Node.js és egy projekt, amin dolgozol. Ennyi.

Telepítsd globálisan

Egyetlen npm parancs, és kész is a CLI.

bash
# telepítés
npm install -g @anthropic-ai/claude-code

Lépj be a projektedbe és indítsd el

Első indításkor bejelentkezel a Claude-fiókoddal. Utána már beszélgethettek.

bash
cd ~/projektem
claude

Mondd el, mit szeretnél

Akár egy bug, akár egy új feature, akár „magyarázd el, hogyan működik ez a modul". Tipp: nagyobb feladatnál kérd meg, hogy előbb készítsen tervet (Plan mode, Shift+Tab).

claude
› Hol kezelődik a bejelentkezés? Készíts tervet, hogyan
  tegyünk hozzá „elfelejtett jelszó" funkciót.
Automatizálás

Hookok

Mik ezek? Saját shell-parancsok, amik automatikusan lefutnak a Claude Code életciklusának adott pontjain — például mielőtt egy eszközt használ, vagy miután szerkesztett egy fájlt.

A lényeg: ez determinisztikus. Nem az AI-ra bízod, hogy „ne felejtse el" lefuttatni a formázót — a hook mindig lefut, mert te állítottad be, nem ő dönt róla.

  • ✓Auto-formázás: Prettier / ESLint minden szerkesztés után, magától.
  • ✓Védőháló: veszélyes parancsok (pl. rm -rf) blokkolása futás előtt.
  • ✓Értesítés: kapj egy hangot vagy üzenetet, ha végzett egy feladattal.
  • ✓Naplózás: jegyezd fel, mit csinált, audithoz vagy tanuláshoz.
PreToolUse PostToolUse UserPromptSubmit Stop SessionStart Notification

Példa: formázz minden fájlt, amit Claude szerkesztett. A .claude/settings.json-ba kerül:

.claude/settings.json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "npx prettier --write \"$CLAUDE_PROJECT_DIR\""
          }
        ]
      }
    ]
  }
}

A hook a tool adatait JSON-ként kapja meg a stdin-en, és kilépési kóddal jelezhet vissza (pl. exit 2 = állítsd le / blokkold a műveletet).

Újrahasználható tudás

Skillek

Mik ezek? Egy skill egy mappa, benne egy SKILL.md fájl, ami leírja, hogyan kell egy adott dolgot csinálni. Lehet mellé szkript, sablon, bármilyen segédfájl.

A varázslat: Claude a leírás (description) alapján magától eldönti, mikor van rá szükség, és előveszi. Olyan, mintha betanítanád egy új kollégát egy folyamatra, és onnantól tudja.

  • ✓Csapat-tudás: „nálunk így csinálunk egy PR-t / egy migrációt / egy release-t".
  • ✓Kész képességek: léteznek pl. PDF-, Word-, Excel-kezelő skillek dobozból.
  • ✓Kétféleképp indul: Claude automatikusan, vagy te kézzel /skill-neve paranccsal.
  • ✓Megosztható: berakod a repóba, és az egész csapat ugyanúgy dolgozik.

Egy egyszerű skill. A fájl helye: .claude/skills/pr-review/SKILL.md

SKILL.md
---
name: pr-review
description: A csapatunk PR-review folyamata.
  Akkor használd, amikor egy pull requestet
  vagy egy diffet kell átnézni.
---

# PR Review folyamat

1. Nézd át a diffet hibákért és biztonsági résekért.
2. Ellenőrizd, hogy van-e teszt az új kódhoz.
3. Nézd meg, követi-e a kód a CONTRIBUTING.md-t.
4. Foglald össze a találatokat súlyosság szerint.

A jó description a kulcs: ebből tudja Claude, mikor érdemes elővennie a skillt.

Külső eszközök bekötése

MCP — Model Context Protocol

Mi ez? Egy nyílt szabvány, amivel az AI-t külső eszközökhöz és adatforrásokhoz kötöd. Gyakran úgy emlegetik: „USB-C az AI-nak" — egy univerzális csatlakozó.

Claude alapból a fájljaidat és a terminált látja. MCP-vel bekötöd a GitHubot, adatbázist, Slacket, böngészőt, Figma-t, Sentryt — bármit, amihez van MCP-szerver. Onnantól ezek eszközeit közvetlenül használja.

  • ✓Valódi adatok: „nyisd meg a #142-es GitHub issue-t és javítsd ki".
  • ✓Lekérdezés: kérdezd a saját adatbázisodat természetes nyelven.
  • ✓Böngésző: nyisson meg egy oldalt és ellenőrizze, működik-e a változtatásod.
  • ✓Szabvány: az Anthropic nyílt forrásúvá tette, ma iparági standard — rengeteg kész szerver van.

MCP-szerver hozzáadása egy paranccsal:

bash
# pl. a GitHub MCP-szerver bekötése
claude mcp add github \
  -- npx -y @modelcontextprotocol/server-github

# bekötött szerverek listája
claude mcp list

Utána egyszerűen kéred, amit akarsz:

claude
› Listázd a nyitott „bug" címkéjű issue-kat,
  és csinálj egy tervet a legrégebbire.
Párhuzamosítás & fókusz

Subagentek

Mik ezek? Specializált al-ágensek, mindegyiknek saját kontextusablaka, saját system promptja és saját eszköz-készlete van. A fő Claude delegál nekik egy-egy feladatot.

Gondolj rá úgy, mint egy csapatvezetőre, aki szétoszt feladatokat. Mindenki a sajátján dolgozik, a vezető pedig csak a kész eredményeket kapja vissza.

  • ✓Tiszta kontextus: egy nagy keresés zaja nem tömi tele a fő beszélgetést — csak a végeredmény jön vissza.
  • ✓Párhuzam: több subagent dolgozhat egyszerre független feladatokon.
  • ✓Specializáció: egy „code-reviewer" agentnek más a prompja és szűkebbek a jogai, mint egy „kutató" agentnek.
  • ✓Beépítettek is vannak: pl. kereső (Explore), tervező (Plan), általános (general-purpose).

Saját subagent definíciója. Helye: .claude/agents/code-reviewer.md

code-reviewer.md
---
name: code-reviewer
description: Kódminőségi és biztonsági
  review-k. Használd proaktívan minden
  nagyobb kódváltoztatás után.
tools: Read, Grep, Glob
---

Tapasztalt code reviewer vagy. Nézd át a
változtatásokat, és keress:
- logikai és határeset-hibákat,
- biztonsági réseket,
- hiányzó teszteket.
A találatokat súlyosság szerint rendezd.

A tools sorral szűkíted, mihez nyúlhat — itt pl. csak olvasni tud, írni nem.

Amire még érdemes ránézni

A négy nagy funkción túl rengeteg apróság teszi kényelmessé a napi használatot.

Slash parancsok

Beépítettek (/init, /clear, /review) és sajátok is, amiket markdown fájlként írsz a .claude/commands/ mappába.

CLAUDE.md — projekt-memória

Egy fájl, amit Claude automatikusan beolvas. Ide írod a konvenciókat, fontos parancsokat, architektúra-jegyzeteket. A /init generál egy kezdőt.

Plan mode

Nagy feladat előtt Claude előbb kutat és tervet készít, fájl-módosítás nélkül. Te jóváhagyod, és csak utána kezd dolgozni. (Shift+Tab)

Engedélyek (permissions)

Te szabod meg, mit tehet rákérdezés nélkül és mihez kell jóváhagyás. Allowlistezhetsz biztonságos parancsokat, hogy ne kérdezzen feleslegesen.

IDE & app integráció

Nemcsak terminál: van VS Code és JetBrains kiterjesztés, valamint asztali és webes app is — ugyanaz az agent, több felületen.

Háttérfeladatok & ütemezés

Hosszan futó parancsokat háttérben futtat, és vissza tud térni rájuk. Sőt, ismétlődő feladatokat is be tudsz ütemezni neki.

Hogyan kapcsolódik ez össze? A CLAUDE.md adja az állandó kontextust, a skillek a folyamat-tudást, az MCP a külső adatokat, a hookok az automata szabályokat, a subagentek pedig a párhuzamos munkaerőt. Együtt egy testreszabott fejlesztői környezetté állnak össze, ami pont úgy dolgozik, ahogy te szeretnéd.

// Pro tippek

Hogyan hozd ki belőle a maximumot

Amit a tapasztalt felhasználók másképp csinálnak.

Írj jó CLAUDE.md-t. Ez a legjobb befektetés: minél jobban ismeri a projekted szabályait, annál pontosabb. Kezdd a /init-tel.
Nagy feladat? Előbb terv. Plan mode-dal átnézed a tervet, mielőtt egy sor kódot is írna. Sokkal kevesebb félreértés.
Bontsd subagentekre. Keresés, review, kutatás — add ki külön agenteknek, hogy a fő beszélgetésed fókuszált maradjon.
Köss be MCP-t a stack-edhez. GitHub, adatbázis, böngésző. Minél többet „lát", annál többet old meg önállóan.
Automatizálj hookokkal. Formázás, lintelés, tesztfuttatás magától — egyszer beállítod, örökre megvan.
Használj /clear-t gyakran. Új, független feladatnál üríts kontextust — gyorsabb és pontosabb lesz.
