# Brainfuck Tetris

Real-time terminal Tetris driven by a **pure Brainfuck** program (`tetris.bf`),
generated by a small DSL → Brainfuck compiler and run by a bundled Python host VM.

## Layout

- `src/oracle.py` — reference pure-Python Brainfuck VM (`run_bf`) used by build/golden tests
- `src/dsl.py` — the `Compiler` (goto-emitter) + verified BF primitive macros
- `src/game.py` — builds the full game using `dsl.py` (emits `tetris.bf`)
- `build.py` — writes `tetris.bf` + `tests/memory_map.txt`, asserts no region overlap
- `bf_run.py` — host VM, terminal I/O, frame loop, sandboxed finale
- `tetris.bf` — GENERATED by `build.py` (gitignored; run `python build.py` to (re)create it)
- `run.cmd` — launches the game

## Build / Test / Run

```
python build.py            # writes tetris.bf + tests/memory_map.txt
python -m pytest tests/ -v # run all tests
python bf_run.py tetris.bf # play (or run.cmd)
```

## The twist (honest)

The well **is** a region of the Brainfuck tape (20×40 = 800 cells, +1 biased).
Tetromino pieces map to Brainfuck command glyphs (`> < + - . [ ]`). Cleared rows
are appended to an assembled, **balance-corrected** Brainfuck program that
actually runs in a sandbox at game over. Balancing may reorder/drop glyphs, so
the finale runs a *valid program derived from your line clears*, not a literal
transcript. Real-time behavior is host-coupled (non-blocking input + ANSI).

## Requirements

Python ≥ 3.11 (high-resolution sleep). Windows console with VT support.

## How it's built

The runtime piece subsystem (scan-locate + relative collision + conditional
move + lock/spawn + sentinel resync) was empirically verified in real Brainfuck
before implementation — see `../docs/superpowers/research/kernel-proofs/`.
