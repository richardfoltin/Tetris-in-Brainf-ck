# Brainfuck Tetris

Real-time terminal Tetris driven by a **pure Brainfuck** program (`tetris.bf`),
generated by a small DSL → Brainfuck compiler and run by a bundled Python host VM.
Playable: drop, move, rotate, lock, line clear, scoring, game over — all from BF.

## Layout

- `src/oracle.py` — reference pure-Python Brainfuck VM (`run_bf`); the build/test truth oracle
- `src/dsl.py` — the `Compiler` (goto-emitter + relative-section mode) + verified BF primitive macros
- `src/game.py` — memory map (region allocator + overlap asserts) and the BF ANSI **renderer**
- `src/subsystem.py` — the verified relative-pointer moving-piece engine (scan/collision/move/lock/spawn)
- `src/driver.py` — input decoder (key byte → one-hot action flags)
- `src/loop.py` — **the game**: feedback bridge, runtime (piece,rot) dispatch, gravity/lock/spawn, line clear, BCD scoring, RNG, main loop, finale
- `build.py` — assembles the full game → writes `tetris.bf` + `tests/memory_map.txt`
- `bf_run.py` — host VM, terminal I/O, atomic per-frame output, frame pacing, finale sandbox (`run_sandbox`, available for the planned twist-B)
- `tetris.bf` — GENERATED by `build.py` (~86 MB, gitignored; run `python build.py` to (re)create it)

## Build / Test / Run

```
python build.py            # writes tetris.bf + tests/memory_map.txt
python -m pytest tests/ -q # run all tests (134)
python bf_run.py tetris.bf # play (first launch ~10s to compile the .bf)
```

**Controls:** `A`/`D` move · `W` rotate · `S` soft-drop · `Q` quit. Use a real
console (Windows Terminal / conhost), not an IDE-embedded one.

## The twist (honest)

- **(A) — done.** The well **is** a region of the Brainfuck tape (20×40 = 800
  cells, +1 biased), and the renderer draws straight from it. Each tetromino maps
  to a Brainfuck command glyph (`> < + - . [ ]`), so locked pieces literally show
  their BF command on the board. You are watching Brainfuck's memory be the well.
- **(B) — planned.** The intended finale assembles the *cleared* rows' glyphs into
  a balance-corrected, runnable Brainfuck program and executes it in a sandbox
  (`bf_run.run_sandbox`) at game over. The sandbox machinery exists; wiring the
  live assembly buffer into the game loop is the remaining piece. Until then the
  finale prints `GAME OVER` + score.

Real-time behavior is host-coupled (non-blocking input + ANSI) — the common
caveat of every real-time BF game.

## Requirements

Python 3.11+ recommended (high-resolution `time.sleep` for steadier pacing); runs
on 3.10 too, just with coarser frame timing. Windows console with VT support.

## How it's built

The runtime piece subsystem (scan-locate + relative collision + conditional
move + lock/spawn + sentinel resync) was empirically verified in real Brainfuck
before implementation — see `../docs/superpowers/research/kernel-proofs/`. The
game loop on top of it (and the problems that surfaced while integrating) is
documented in the root [`../README.md`](../README.md).
